Technical Explanation

This program demonstrates the principle of encapsulation, a core OOP concept that involves hiding internal object details and controlling access to properties. In the UML class diagram, a minus sign (-) means a property or method is private, while a plus sign (+) means it is public. The key idea is that external code should not work directly with hidden data. Instead, it should use the public methods provided by the class. In this example, getVenue() is used to control access to the venue value. Before returning the venue, it checks that the text is between 5 and 15 characters. If the stored value is outside that range, the method returns an error message instead of exposing invalid data.

Design

Class name: Event In the UML class diagram for this program, the properties marked with - are private and the constructor and methods marked with + are public. This is the key change introduced by encapsulation: data is hidden inside the class, and access is controlled through methods.

This example focuses on the venue property. It is not accessed directly from outside the class. Instead, the public method getVenue() checks whether the stored venue is a sensible length before returning it. This shows how encapsulation allows the class to enforce rules about its own data. ### English Pseudocode Constructor: Event() 1. Accept the starting date, time, venue and reminder values. 2. Store each value inside the object as private data. 3. Create an empty participants list. 4. Set the participant index to the first free position. Method: getVenue() 1. Read the private venue value stored in the object. 2. Check whether the venue length is between 5 and 15 characters. 3. If it is valid, return the venue. 4. Otherwise, return an error message. Method: updateDate() 1. Accept a new date value. 2. Change the private start date stored in the object. Method: getDate() 1. Read the private start date from the object. 2. Return that value. Method: getParticipants() 1. Read the participants list stored in the object. 2. Return the list. Method: getParticipantCount() 1. Read the current participant index. 2. Return the number of stored participants. Method: addParticipant() 1. Accept a participant name. 2. Store the name in the participants list at the next free position. 3. Increase the private index so the next participant is stored in the next space.

Python Code
class Event:
    def __init__(self, startDate, startTime, venue, reminder):
        self.__startDate = str(startDate)  #String
        self.__startTime = str(startTime)  #String
        self.__venue = str(venue)          #String
        self.__reminder = reminder if isinstance(reminder, bool) else str(reminder).strip().lower() in ("true", "1", "yes", "y")    #Boolean (don't use bool(reminder): bool("False") is True)
        self.__participants = [""]*20 #Array of String
        self.__index = int(0)              #Integer

    def updateDate(self, eventDate):
        self.__startDate = str(eventDate)

    def getDate(self):
        return self.__startDate

    def getVenue(self):
      if 5 <= len(self.__venue) <= 15:
        return self.__venue
      return "Invalid venue"

    def getParticipants(self):
        return self.__participants

    def getParticipantCount(self):
        return self.__index

    def addParticipant(self, name):
        self.__participants[self.__index] = str(name)
        self.__index = self.__index + 1
