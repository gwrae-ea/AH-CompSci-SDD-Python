Technical Explanation

This program builds directly on Program 5 by testing the encapsulated Event class in more depth. The focus is no longer just on making properties private: it is on proving that the class protects its data correctly. In particular, the program tests the validation inside getVenue(), which only returns the venue when it is between 5 and 15 characters long. It also checks that valid operations still work, that invalid values produce the expected message, that object state remains consistent after testing, and that private attributes should be accessed through methods rather than directly.

Analysis

The user wants to see that encapsulation actually prevents data corruption. 4. Design

Class name: Event This program keeps the encapsulated class from Program 5. The UML class diagram still shows the same encapsulated design, with - for private properties and + for public methods. The focus of this program is testing that design, but the design itself should still describe the methods in the class.

### English Pseudocode Constructor: Event() 1. Accept the starting date, time, venue and reminder values. 2. Store each value inside the object as private data. 3. Create an empty participants list. 4. Set the participant index to the first free position. Method: getVenue() 1. Read the private venue value stored in the object. 2. Check whether the venue length is between 5 and 15 characters. 3. If it is valid, return the venue. 4. Otherwise, return an error message. Method: updateDate() 1. Accept a new date value. 2. Change the private start date stored in the object. Method: getDate() 1. Read the private start date from the object. 2. Return that value. Method: addParticipant() 1. Accept a participant name. 2. Store the name in the participants list at the next free position. 3. Increase the private index so the next participant is stored in the next space.

Python Code
class Event:
    def __init__(self, startDate, startTime, venue, reminder):
        self.__startDate = str(startDate)  #String
        self.__startTime = str(startTime)  #String
        self.__venue = str(venue)          #String
        self.__reminder = reminder if isinstance(reminder, bool) else str(reminder).strip().lower() in ("true", "1", "yes", "y")    #Boolean (don't use bool(reminder): bool("False") is True)
        self.__participants = [""]*20 #Array of String
        self.__index = int(0)              #Integer

    def updateDate(self, eventDate):
        self.__startDate = str(eventDate)

    def getDate(self):
        return self.__startDate

    def getVenue(self):
        if 5 <= len(self.__venue) <= 15:
            return self.__venue
        return "Invalid venue"

    def getParticipants(self):
        return self.__participants

    def getParticipantCount(self):
        return self.__index

    def addParticipant(self, name):
        self.__participants[self.__index] = str(name)
        self.__index = self.__index + 1

# Valid venue test
item1 = Event("13/04/18", "0900", "Main Hall", True)
print("Valid venue:", item1.getVenue())

# Too-short venue test
item2 = Event("13/04/18", "0900", "Gym", True)
print("Too-short venue:", item2.getVenue())

# Too-long venue test
item3 = Event("13/04/18", "0900", "International Conference Centre", True)
print("Too-long venue:", item3.getVenue())

# Date update test
print("Original date:", item1.getDate())
item1.updateDate("14/04/22")
print("Updated date:", item1.getDate())

# Participant tests
item1.addParticipant("Erica Knowles")
item1.addParticipant("Norman Osborne")
print("Participant count:", item1.getParticipantCount())
print("Participants:", item1.getParticipants())

# Direct access test
try:
    print(item1.__venue)
except AttributeError:
    print("Direct access blocked; use getVenue() instead")
