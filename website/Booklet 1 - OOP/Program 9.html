Technical Explanation

This program provides the starter code for the next task. It keeps the Event and WorkMeeting classes from Program 8 so that the learner can begin with a working inheritance and overriding example. In Task 9, this starter will be adapted to create a new child class called Personal. The learner will keep the inherited event behaviour, create a new overridden method, and add suitable private properties with getter and setter methods.

Analysis

### End User Requirements 1. The user wants starter code that already demonstrates inheritance and overriding. 2. The user wants to reuse the existing Event design when creating a new subclass. 3. The user wants a clear example that can be adapted into a personal-event class. 4. The user wants to add new child-class data and methods without changing the whole parent class. ### Functional Requirements #### Advanced Higher concepts The solution is required to: FR1 Provide a working parent class called Event. FR2 Provide a working child class called WorkMeeting that inherits from Event. FR3 Demonstrate an overridden getDate() method in the child class. FR4 Give a valid starter structure that can be adapted into a new Personal subclass. #### Integration The solution is required to: FR5 Show how super() is used to call the parent constructor. FR6 Include child-specific properties and methods in the starter code. FR7 Display output that proves the overridden method works. #### Additional functional requirements The solution is required to: FR8 Create at least one child-class object using realistic test data. FR9 Show that the child object still uses inherited behaviour from Event. FR10 Keep the starter code simple enough to be modified in Task 9. FR11 Prepare the learner to add getter and setter methods in the next task. ---

Design

Parent class: Event Child class: WorkMeeting This starter keeps the Program 8 class structure. The learner can see how the child class inherits the common event information from Event, overrides getDate(), and adds new child-specific properties and methods. The same pattern will then be reused in Task 9 to create a Personal class. ### English Pseudocode Parent Constructor: Event() 1. Accept the starting date, time, venue and reminder values. 2. Store each value inside the event object. 3. Create an empty participants list. 4. Set the participant index to the first free position. Parent Method: getDate() 1. Read the stored date from the object. 2. Return that date value. Child Constructor: WorkMeeting() 1. Accept all the event details plus the meeting title and mileage. 2. Call the parent constructor to set up the inherited event data. 3. Store the child-specific details in the child object. 4. Set the initial travel expenses to zero. Child Method: getDate() 1. Call the parent version of getDate(). 2. Add text that makes the message specific to a work meeting. 3. Return the new combined string. Child Method: calculateTravelExpenses() 1. Read the mileage stored in the child object. 2. Multiply the mileage by 1.5. 3. Store the result as the travel expenses. Child Method: getTravelExpenses() 1. Read the stored travel expenses. 2. Return that value.

Implementation

[Program 9 - personal subclass starter.py](./Program%209%20%E2%80%93%20personal%20subclass%20starter.py) SQA-RL: ``` CLASS Event PRIVATE startDate AS STRING PRIVATE startTime AS STRING PRIVATE venue AS STRING PRIVATE reminder AS BOOLEAN PRIVATE participants AS ARRAY[1:20] OF STRING PRIVATE index AS INTEGER PROCEDURE Event(newStartDate, newStartTime, newVenue, newReminder) SET startDate TO newStartDate SET startTime TO newStartTime SET venue TO newVenue SET reminder TO newReminder SET index TO 0 END PROCEDURE FUNCTION getDate() RETURNS STRING RETURN startDate END FUNCTION END CLASS CLASS WorkMeeting EXTENDS Event PRIVATE meetingTitle AS STRING PRIVATE mileage AS REAL PRIVATE travelExpenses AS REAL PROCEDURE WorkMeeting(newStartDate, newStartTime, newVenue, newReminder, newMeetingTitle, newMileage) CALL SUPER.Event(newStartDate, newStartTime, newVenue, newReminder) SET meetingTitle TO newMeetingTitle SET mileage TO newMileage SET travelExpenses TO 0 END PROCEDURE FUNCTION getDate() RETURNS STRING RETURN "Work meeting on " + SUPER.getDate() END FUNCTION PROCEDURE calculateTravelExpenses() SET travelExpenses TO mileage * 1.5 END PROCEDURE FUNCTION getTravelExpenses() RETURNS REAL RETURN travelExpenses END FUNCTION END CLASS DECLARE workMeeting1 AS NEW WorkMeeting("14/04/21", "0900", "Main Office", TRUE, "Work from Home", 7.5) SEND workMeeting1.getDate() TO DISPLAY CALL workMeeting1.calculateTravelExpenses() SEND workMeeting1.getTravelExpenses() TO DISPLAY ``` ### Notes - This starter repeats the Program 8 inheritance and overriding structure - The child version reuses the parent version with super() - Overriding allows more specific behaviour in the child class - The same structure can be adapted in Task 9 to create a Personal class with getters and setters ---
Python Code
class Event:
        def __init__(self, startDate, startTime, venue, reminder):
          self.__startDate = str(startDate)  #String
          self.__startTime = str(startTime)  #String
          self.__venue = str(venue)          #String
          self.__reminder = reminder if isinstance(reminder, bool) else str(reminder).strip().lower() in ("true", "1", "yes", "y")    #Boolean (don't use bool(reminder): bool("False") is True)
          self.__participants = [""]*20 #Array of String
          self.__index = int(0)              #Integer

        def updateDate(self, eventDate):
          self.__startDate = str(eventDate)

        def getDate(self):
          return self.__startDate

        def addParticipant(self, name):
          self.__participants[self.__index] = str(name)
          self.__index = self.__index + 1

      class WorkMeeting(Event):
        def __init__(self, startDate, startTime, venue, reminder, meetingTitle, mileage):
          super().__init__(startDate, startTime, venue, reminder)
          self.__meetingTitle = str(meetingTitle)      #String
          self.__mileage = float(mileage)              #Real
          self.__travelExpenses = float(0)             #Real

        def getDate(self):
          return "Work meeting on " + super().getDate()

        def calculateTravelExpenses(self):
          self.__travelExpenses = self.__mileage * 1.5
    
        def getTravelExpenses(self):
          return self.__travelExpenses

      # Test the overridden getDate() method.
      workMeeting1 = WorkMeeting(startDate = "14/04/21", startTime = "0900", venue = "Main Office", reminder = True, meetingTitle = "Work from Home", mileage = 7.5)

      print("The date details are", workMeeting1.getDate())
      workMeeting1.calculateTravelExpenses()
      print("Travel expenses are £", workMeeting1.getTravelExpenses())
