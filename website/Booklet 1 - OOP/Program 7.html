Technical Explanation

This program demonstrates inheritance, a fundamental OOP principle where a new class (child/derived class) inherits properties and methods from an existing class (parent/base class). Inheritance allows code reuse and establishes logical hierarchies. In this example, `Event` is the parent class and `WorkMeeting` is the child class. `WorkMeeting` inherits event behavior and adds meeting-specific properties such as `meetingTitle`, `mileage`, and methods for travel expense calculation. The program demonstrates how the child class extends the parent, how to call parent class methods using `super()`, and how inherited and child-specific behavior can be used together.

Design

Parent class: Event Child class: WorkMeeting This program introduces inheritance. Inheritance allows a new class to be created from an existing class. The new class is called the child class and it automatically receives the properties and methods of the parent class. In this diagram, WorkMeeting inherits from Event, so it can use the event data and methods already defined in the parent class, while also adding its own extra properties and methods.

### English Pseudocode Parent Constructor: Event() 1. Accept the starting date, time, venue and reminder values. 2. Store each value inside the event object. 3. Create an empty participants list. 4. Set the participant index to the first free position. Parent Method: getVenue() 1. Read the venue stored in the parent object. 2. Return that venue value. Parent Method: updateDate() 1. Accept a new date value. 2. Replace the current stored date with the new date. Parent Method: getDate() 1. Read the stored date from the object. 2. Return that date value. Parent Method: addParticipant() 1. Accept a participant name. 2. Store the name in the participants list at the next free position. 3. Increase the index so the next participant is stored in the next space. Child Constructor: WorkMeeting() 1. Accept all the event details plus the meeting title and mileage. 2. Call the parent constructor to set up the inherited event data. 3. Store the meeting title and mileage in the child object. 4. Set the initial travel expenses to zero. Child Method: calculateTravelExpenses() 1. Read the mileage stored in the child object. 2. Multiply the mileage by 1.5. 3. Store the result as the travel expenses. Child Method: getTravelExpenses() 1. Read the stored travel expenses. 2. Return that value.

Implementation

[Program 7 - inheritance.py](./Program%207%20%E2%80%93%20inheritance.py) SQA-RL: ``` CLASS Event PRIVATE startDate AS STRING PRIVATE startTime AS STRING PRIVATE venue AS STRING PRIVATE reminder AS BOOLEAN PRIVATE participants AS ARRAY[1:20] OF STRING PRIVATE index AS INTEGER PROCEDURE Event(newStartDate, newStartTime, newVenue, newReminder) SET startDate TO newStartDate SET startTime TO newStartTime SET venue TO newVenue SET reminder TO newReminder SET index TO 0 END PROCEDURE FUNCTION getVenue() RETURNS STRING RETURN venue END FUNCTION PROCEDURE updateDate(eventDate) SET startDate TO eventDate END PROCEDURE FUNCTION getDate() RETURNS STRING RETURN startDate END FUNCTION PROCEDURE addParticipant(name) SET participants[index] TO name SET index TO index + 1 END PROCEDURE END CLASS CLASS WorkMeeting EXTENDS Event PRIVATE meetingTitle AS STRING PRIVATE mileage AS REAL PRIVATE travelExpenses AS REAL PROCEDURE WorkMeeting(newStartDate, newStartTime, newVenue, newReminder, newMeetingTitle, newMileage) CALL SUPER.Event(newStartDate, newStartTime, newVenue, newReminder) SET meetingTitle TO newMeetingTitle SET mileage TO newMileage SET travelExpenses TO 0 END PROCEDURE PROCEDURE calculateTravelExpenses() SET travelExpenses TO mileage * 1.5 END PROCEDURE FUNCTION getTravelExpenses() RETURNS REAL RETURN travelExpenses END FUNCTION END CLASS ``` Inheritance is shown in the full class definitions above. WorkMeeting extends Event, so it reuses the parent constructor and methods, then adds its own properties and behaviour for work meetings. ### Notes - Parent class is the base; child class builds on it - Child class inherits without modification - Child can override parent methods - `super()` calls parent implementation ---
Python Code
class Event:
    def __init__(self, startDate, startTime, venue, reminder):
        self.__startDate = str(startDate)  #String
        self.__startTime = str(startTime)  #String
        self.__venue = str(venue)          #String
        self.__reminder = reminder if isinstance(reminder, bool) else str(reminder).strip().lower() in ("true", "1", "yes", "y")    #Boolean (don't use bool(reminder): bool("False") is True)
        self.__participants = [""]*20 #Array of String
        self.__index = int(0)              #Integer

    def updateDate(self, eventDate):
          self.__startDate = str(eventDate)

    def getDate(self):
          return self.__startDate

        def getVenue(self):
          return self.__venue

    def addParticipant(self, name):
          self.__participants[self.__index] = str(name)
          self.__index = self.__index + 1

class WorkMeeting(Event):
        def __init__(self, startDate, startTime, venue, reminder, meetingTitle, mileage):
        super().__init__(startDate, startTime, venue, reminder)
          self.__meetingTitle = str(meetingTitle)      #String
          self.__mileage = float(mileage)              #Real
          self.__travelExpenses = float(0)             #Real

    def calculateTravelExpenses(self):
          self.__travelExpenses = self.__mileage * 1.5
    
    def getTravelExpenses(self):
          return self.__travelExpenses
